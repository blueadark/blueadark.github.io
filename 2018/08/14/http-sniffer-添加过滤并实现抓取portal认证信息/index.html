http-sniffer 添加过滤并实现抓取portal认证信息

发表于 2018-08-14

主要工作：

  1. 对http嗅探器进行修改，该嗅探器以往只对http报头进行分析，获取http报头中请求信息的头部和回复信息的头部的内容，经过更改将http的body正文输出。
  2. 增加其抓包过滤过滤功能实现对只符合条件的数据包的过滤。
    3.抓取portal认证信息

#介绍：#

http-sniffer：
是一种多线程工具，可以用于从pcap文件中嗅探TCP流统计信息和嵌入式HTTP标头。每个携带HTTP的TCP流都以json格式导出到文本中。

portal认证：
Portal认证的基本过程是：客户机首先通过DHCP协议获取到IP地址（也可以使用静态IP地址），但是客户使用获取到的IP地址并不能登上Internet，在认证通过前只能访问特定的IP地址，这个地址通常是PORTAL服务器的IP地址。采用Portal认证的接入设备必须具备这个能力。一般通过修改接入设备的访问控制表（ACL）可以做到。

用户登录到Portal Server后，可以浏览上面的内容，比如广告、新闻等免费信息，同时用户还可以在网页上输入用户名和密码，它们会被WEB客户端应用程序传给 Portal Server，再由Portal Server与NAS之间交互来实现用户的认证。

Portal Server在获得用户的用户名和密码外，还会得到用户的IP地址，以它为索引来标识用户。然后Portal Server 与NAS之间用Portal协议直接通信，而NAS又与RADIUS 服务器直接通信完成用户的认证和上线过程。因为安全问题，通常支持安全性较强的CHAP式认证。

优点：

  • 不需要特殊的客户端软件，降低网络维护工作量

  • l可以提供Portal等业务认证

缺点：

  • WEB承载在7层协议上，对于设备的要求较高，建网成本高；

  • 用户连接性差，不容易检测用户离线，基于时间的计费较难实现；

  • 易用性不够好，用户在访问网络前，不管是 TELNET、FTP还是其它业务，必须使用浏览器进行WEB认证；

  • IP地址的分配在用户认证前，如果用户不是上网用户，则会造成地址的浪费，而且不便于多ISP的支持。

  • 认证前后业务流和数据流无法区分

portal无感知认证：
简单的来说就是可以避免每次连接网络都需要进行portal认证，避免每次输入用户名及密码带来的繁琐
具体操作是将你当前设备的mac地址与你的portal认证用户名相绑定

    main.c函数中的capture_main（）函数 

    int capture_main(const char* interface, void (*pkt_handler)(void*), int livemode,char *rules){
char errbuf[PCAP_ERRBUF_SIZE];
memset(errbuf, 0, PCAP_ERRBUF_SIZE);
char *raw = NULL;
pcap_t *cap = NULL;
struct bpf_program filter;
struct pcap_pkthdr pkthdr;
packet_t *packet = NULL;
bpf_u_int32 net;
bpf_u_int32 mask;
extern int GP_CAP_FIN;
// printf("%s mode ...\n", livemode==1 ? "Online" : "Offline");
printf("%s\n",interface);
printf("%s",rules);
if ( livemode==1 ) {
pcap_lookupnet(interface,&net,&mask,errbuf);
    cap = pcap_open_live(interface, 65535, 0, 1000, errbuf);
} else {
    cap = pcap_open_offline(interface, errbuf);
}
if( cap == NULL) {
    printf("%s\n",errbuf); exit(1);
}
while(1){
pcap_compile(cap,&filter,rules,0,net);
pcap_setfilter(cap,&filter);
    raw = pcap_next(cap, &pkthdr);
    if( NULL != raw){
        packet = packet_preprocess(raw, &pkthdr);
        if( NULL != packet ){
            pkt_handler(packet);
        }
    } else if ( livemode==0 ) {
        GP_CAP_FIN = 1;
        break;
    } else {
        nanosleep((const struct timespec[]){{0, 100000000L}}, NULL);
    }
}

if( cap != NULL)
    pcap_close(cap);

return 0;

}

    主要是应用libpcap的pcap抓包框架：

pcap_t pcap_open_live(char device, int snaplen, int promisc, int to_ms, char ebuf)

函数功能：是获得数据包捕获描述字的函数

  • snaplen参数定义捕获数据的最大字节数， 一般使用BUFSIZ， 该参数一般位于<pcap.h>中，若没有定义，应使用unsigned int的最大值。
  • promisc指定是否将网络接口置于混杂模式， 如果设置为true(当promisc>0时)可以使用混杂模式进行数据包的抓取。
  • to_ms参数指定超时时间（毫秒），如果设置为0意味着没有超时等待这一说法。
  • ebuf参数则仅在pcap_open_live()函数出错返回NULL时用于传递错误消息。

pcap_t pcap_open_offline(char fname,char *errbuf)

函数功能：获得保存在fname文件中的以前捕捉的数据。如果出错，错误保存在errbuf中。

const u_char pcap_next(pcap_t p, struct pcap_pkthdr *h)

函数功能：返回一个无符号指针指向抓取到的数据，参数h为包的一些基本信息
然后调到packet_preprocess()函数

int pcap_compile(pcap_t p, strUCt bpf_program fp, char *str, int optimize, bpf_u_int32 netmask)

函数功能：设置并编译过滤器

  • *p是存放的会话句柄
  • *fp是存放被编译的过滤器的版本的引用
  • *str是存放过滤表达式
  • optimize定义表达式是否被优化的整形量（0为false，1为true，标准规定）
  • netmask是网络掩码

int pcap_setfilter(pcap_t p, struct bpf_program fp)

函数功能：使用过滤器

  • *p存放会话句柄
  • *fp存放被编译的过滤器的版本的引用

  • ##数据包分析

    main.c中的packet__preprocess()函数。packet_ _preprocess主要是解析数据包头部的信息的函数，用来解析以太网帧首部，ip帧首部，tcp帧首部，http报文首部的信息。： 

    packet_t *packet_preprocess(const char *raw_data, const struct pcap_pkthdr *pkthdr)
{
packet_t    *pkt = NULL;    /* new packet */
const char     *cp = raw_data;
ethhdr    *eth_hdr = NULL;
iphdr    *ip_hdr = NULL;
tcphdr    *tcp_hdr = NULL;

/* Parse libpcap packet header */
pkt = packet_new();
pkt->cap_sec = pkthdr->ts.tv_sec;
pkt->cap_usec = pkthdr->ts.tv_usec;
pkt->raw_len = pkthdr->caplen;

/* Parse ethernet header and check IP payload */
eth_hdr = packet_parse_ethhdr(cp);
if(eth_hdr->ether_type != 0x0800){
    free_ethhdr(eth_hdr);
    packet_free(pkt);
    return NULL;
}

/* Parse IP header and check TCP payload */
cp = cp + sizeof(ethhdr);
ip_hdr = packet_parse_iphdr(cp);
pkt->saddr = ip_hdr->saddr;
pkt->daddr = ip_hdr->daddr;
pkt->ip_hl = (ip_hdr->ihl) << 2;    /* bytes */
pkt->ip_tol = ip_hdr->tot_len;
pkt->ip_pro = ip_hdr->protocol;
if(pkt->ip_pro != 0x06){
    free_ethhdr(eth_hdr);
    free_iphdr(ip_hdr);
    packet_free(pkt);
    return NULL;
}

/* Parse TCP header */
cp = cp + pkt->ip_hl;
tcp_hdr = packet_parse_tcphdr(cp);
pkt->sport = tcp_hdr->th_sport;
pkt->dport = tcp_hdr->th_dport;
pkt->tcp_seq = tcp_hdr->th_seq;
pkt->tcp_ack = tcp_hdr->th_ack;
pkt->tcp_flags = tcp_hdr->th_flags;
pkt->tcp_win = tcp_hdr->th_win;
pkt->tcp_hl = tcp_hdr->th_off << 2;        /* bytes */
pkt->tcp_dl = pkt->ip_tol - pkt->ip_hl - pkt->tcp_hl;
pkt->http = 0; /* default */

/* Check the TCP ports to identify if the packet carries HTTP data
   We only consider normal HTTP traffic without encryption */
if( !(tcp_hdr->th_sport == 80 || tcp_hdr->th_dport == 80 || \
    tcp_hdr->th_sport == 8080 || tcp_hdr->th_dport == 8080 || \
    tcp_hdr->th_sport == 8000 || tcp_hdr->th_dport == 8000)){
    free_ethhdr(eth_hdr);
    free_iphdr(ip_hdr);
    free_tcphdr(tcp_hdr);
    packet_free(pkt);
    return NULL;
}

/* Process packets of flows which carry HTTP traffic */
if(pkt->tcp_dl != 0){
    cp = cp + pkt->tcp_hl;
    if( !IsHttpPacket(cp, pkt->tcp_dl) ){
        /* If the packet is not HTTP, we erase the payload. */
        pkt->tcp_odata = NULL;
        pkt->tcp_data = pkt->tcp_odata;
    }else{
        /* Yes, it's HTTP packet */
        char *head_end = NULL;
        int hdl = 0;

    int bodylength=0;

    head_end = IsRequest(cp, pkt->tcp_dl);
if( head_end != NULL ){
    /* First packet of request. */
    hdl = head_end - cp + 1;
    pkt->http = HTTP_REQ;

    bodylength=pkt->tcp_dl-hdl;

    /* Fake TCP data length with only HTTP header. */
pkt->tcp_dl = hdl;

    pkt->http_dl=bodylength;

    }
head_end = IsResponse(cp, pkt->tcp_dl);
if( head_end != NULL ){
    /* First packet of response. */
    hdl = head_end - cp + 1;
    pkt->http = HTTP_RSP;

    bodylength=pkt->tcp_dl-hdl;

    /* Fake TCP data length with only HTTP header. */
pkt->tcp_dl = hdl;

    pkt->http_dl=bodylength;

    }
/* Allocate memory to store HTTP header. */
pkt->tcp_odata = MALLOC(char, pkt->tcp_dl + 1);
pkt->tcp_data = pkt->tcp_odata;
memset(pkt->tcp_odata, 0, pkt->tcp_dl + 1);
memcpy(pkt->tcp_odata, cp, pkt->tcp_dl);

    /Allocate memory to store HTTP body./
    pkt->http_odata=MALLOC(char,pkt->http_dl+1);
    pkt->http_data=pkt->http_odata;
    memset(pkt->http_odata,0,pkt->http_dl+1);
    memcpy(pkt->http_odata,cp+pkt->tcp_dl,pkt->http_dl);

        }
}else{
    pkt->tcp_odata = NULL;
    pkt->tcp_data = pkt->tcp_odata;

    pkt->http_odata= NULL;
    pkt->http_data=pkt->http_odata;

    }
free_ethhdr(eth_hdr);
free_iphdr(ip_hdr);
free_tcphdr(tcp_hdr);
return pkt;

    }

  1. 解析libpcap包的头部信息
    pkt->cap_sec = pkthdr->ts.tv_sec; //时间戳信息
    pkt->cap_usec = pkthdr->ts.tv_usec; //时间戳信息

  1. 解析以太网ethernet头部信息
    LINUX用eth_ hdr用来表示以太网帧的头部，packet_ parse_ ethhdr为分析以太网帧的函数

    hdr->ether_ dhost即以太网帧的目的mac地址，hdr->ether_ shost即以太网帧的源mac地址，hdr->ether_type即网络层使用的协议类型

通过判断hdr->ether_type是否为0x0800来判断以太网帧是否携带ip帧字段

  1. 解析ip帧头部信息
    cp = cp + sizeof(ethhdr); 将以太网帧转到ip帧
    packet_ parse_ iphdr()为分析ip帧的函数

    hdr->ihl即为ip帧的报头长度
    hdr->version即为ip协议实现的版本号
    hdr->tos即为服务类型
    hdr->tot_len即整个ip帧数据报的长度
    hdr->id即标识，16位，用来唯一地标识主机发送的每一份数据报
    hdr->frag _off即段偏移量，13位，指出较长的分组在分段后，某段在原分组的相对位置
    hdr->ttl生存期（TTL），8位，用来设置数据报最多可以经过的路由器数
    hdr->protocol即ip帧数据字段中的数据采用的是上层的哪种协议，常见的有ICMP（1）、IGMP（2）、TCP（6）、UDP（17）
    hdr->check即头部校验和，16位，填充根据IP头部计算得到的校验和码
    hdr->saddr即源ip地址
    hdr->daddr即目的ip地址

通过判断hdr->protocol是否为0x06来判断是否携带tcp帧字段

  1. 解析tcp帧头部信息
    cp=cp+pkt->ip_hl; 将ip帧转到tcp帧，ip_hl指ip帧的头部长，cp是指针
    packet_ parse_tcphdr(cp)为分析tcp帧的头部信息

    hdr->th _sport即源端口
    hdr->th _dport即目的端口
    hdr->th _seq即顺序号，用来表示从tcp源端向tcp目的端发送的数据字节流
    hdr->th _ack即确认号，用来表示发送确认的一端所期望收到的下一个顺序号
    hdr->th _off即报头长度
    hdr->th _flags控制位
    hdr->th _win窗口大小，2个字节，表示从确认号开始，本报文的源方可以接收的字节数，即源方接收窗口大小
    hdr->th _sum校验和，2个字节，对整个的 TCP 报文段（包括 TCP 头部和 TCP 数据），以 16 位字进行计算所得
    hdr->th _urp紧急指针

    通过判断端口是否为80/8080/8000来判断是否携带http字段

  2. 解析http报文对信息

    cp=cp+pkt->tcp_hl; 将tcp帧转到http报文，tcp->hl指tcp帧的头部长，cp是指针
    IsHttpPacket(cp,pkt->tcp _dl)函数功能即判断是否是http包

    这里调用两个函数IsRequest()和IsResponse()来分别判断是否携带http请求报文和http响应报文
    并返回http报头长度


    http _request _method为获取http中的请求方法；常见的method为：GET,HEAD,POST,PUT,DELETE,CONNECT,OPTIONS,TRACE,PATCH,MOVE,COPY,LINK,UNLINK,WRAPPED,Extension-mothed等
    find _header _end()函数为查找到http头部结束位置，也就是http-body开始的位置，
    具体是通过判断连续两个\n的位置，因为\r\n\r\n是头部和主体区分的部分
    http _response _version()函数即判断响应时响应行中的http协议版本，通常是8字节，常见的例如”HTTP/1.0”和”HTTP/1.1”

    从而得出：
    pkt _http=HTTP _REQ或HTTP _RSP; 报文类型
    pkt->http _dl=pkt->tcp _dl-hdl; 在这里我们取http报文body长度
    pkt->tcp _dl = hdl； 在这里我们取http报头head长度

strpd(char *p,char *q)
{
int lenp=strlen(p);
int lenq=strlen(q);
int i;
for (i=0;i<=lenp-lenq;i++)
{
 if (strncmp(p+i,q,lenq)==0) return i;
 }
return -1;
 }
